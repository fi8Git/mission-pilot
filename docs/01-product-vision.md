# 01 — Product Vision

## Vision

MissionPilot helps freelance developers run their mission search like a professional sales pipeline while staying ethical, local-first, and fully in control.

The product does not replace human judgment. It removes repetitive work and supports better decision-making.

## Positioning

MissionPilot is not a job board and not a recruitment automation bot.

It is a personal mission-search cockpit:

- opportunity tracker;
- fit scorer;
- message assistant;
- CV selector;
- follow-up planner;
- recruiter CRM;
- portfolio-grade open-source project.

## Why this project matters

Freelancers often treat mission search as an improvised process:

- opportunities are scattered across platforms;
- recruiters are contacted inconsistently;
- messages are rewritten from scratch;
- poor-fit missions consume time;
- follow-ups are forgotten;
- no data is kept on which channels work best.

MissionPilot turns this into a repeatable workflow.

## Product promise

> Spend less time managing the search process and more time having relevant conversations.

## Differentiation

### Compared to a spreadsheet

MissionPilot provides business rules, message templates, scoring, structured statuses, and a professional dashboard.

### Compared to a CRM

MissionPilot is lightweight, local-first, developer-focused, and does not require account management or cloud setup.

### Compared to outreach automation tools

MissionPilot is ethical and human-controlled. It does not scrape platforms or send automated messages.

## Product principles

### 1. Local-first

The user owns the data. The first version stores data locally.

### 2. Human validation

The application can prepare messages, but the user decides what to send.

### 3. Responsible automation

Automation is used for analysis, draft generation, reminders, and organization—not for spam or platform abuse.

### 4. Recruiter-friendly

The tool should encourage better messages, better matching, and fewer irrelevant applications.

### 5. Developer portfolio quality

The project must be written and documented as if a recruiter, CTO, or technical interviewer may inspect it.

## Main workflows

### Opportunity intake

The user creates or imports an opportunity and stores the key information.

### Opportunity assessment

The system evaluates the opportunity using transparent scoring rules.

### Application preparation

The system suggests:

- priority;
- risk;
- missing skills;
- CV profile;
- message template.

### Application tracking

The user tracks each opportunity through clear statuses.

### Follow-up management

The system highlights candidates and recruiters that need follow-up.

### Learning from results

The user can identify which channels and message types produce interviews.

## Expected user value

- Faster opportunity qualification.
- Better prioritization.
- More consistent communication.
- Fewer missed follow-ups.
- Clear history of recruiter conversations.
- Better focus on high-fit missions.
- A public project that supports professional credibility.

## MVP product boundaries

The MVP focuses on manual entry and structured assistance.

A mission can be added manually by copy-pasting the announcement. The system extracts no data automatically in the first version unless the user enters it directly.

## Future product direction

Possible extensions:

- local AI scoring and message generation;
- import from email alerts;
- export a weekly search report;
- recruiter relationship tracking;
- reusable CV profiles;
- integration with local calendar reminders;
- optional cloud sync for private use.
